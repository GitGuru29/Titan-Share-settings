ArchTitan Settings - Known Problems & Solutions

1. AC Power State Detection
- Problem: The power widget incorrectly showed "On Battery" when plugged into AC power but not actively charging (e.g., when fully charged or restricted by battery protection thresholds).
- Solution: Added a dedicated `acConnected` property to the `SystemInfo` C++ backend to parse the `online` state from `/sys/class/power_supply/AC*`. Updated `PowerPage.qml` to conditionally display "đź”Ś Plugged In" based on this hardware state rather than relying solely on the active charging status.

2. Hardware-Specific Battery Management
- Problem: The battery management UI needed to dynamically adapt to different manufacturer hardware (ThinkPad, ASUS, Lenovo IdeaPad) while handling missing sysfs write permissions gracefully.
- Solution: Implemented `SystemInfo.chargeProtectionMode` to dynamically render the appropriate UI components in QML (threshold sliders for ASUS/Thinkpad, binary toggles for IdeaPads). Created `install-battery-rules.sh` to deploy udev rules that grant standard users write access to battery sysfs files.

3. Power Profile Cycling Shortcut
- Problem: The keyboard shortcut to cycle power profiles was failing due to broken system dependencies in the desktop environment.
- Solution: Created `scripts/cycle_power.sh` that utilizes direct D-Bus communication to bypass the broken layers, reliably cycling between Power Saver, Balanced, and Performance modes.

4. Incorrect/Missing UI Branding Assets
- Problem: Generic or corrupted icons (like the speedometer) were rendering in the power profiles and About page, and `rebuild.sh` was not handling assets correctly.
- Solution: Replaced placeholders with high-quality, 3D neon assets and the correct `archtitan-logo.png`. Fixed the Qt Resource System configuration and updated QML to utilize `MultiEffect` for correct accent colorization.

5. Missing System Information Data
- Problem: The CPU model and RAM usage were rendering as "unknown" or empty in the interface.
- Solution: Fixed the backend `readMemory` and `initCpuModel` routines to accurately parse `/proc/cpuinfo` and `/proc/meminfo`, establishing reactive QML properties that update the UI on interval.

6. Settings Persistence Issues
- Problem: Changes to accent colors, icon themes, and fonts were resetting after closing the app or rebooting.
- Solution: Debugged the `applyAndSave()` logic in `SettingsBackend` to ensure system-level persistence via `gsettings` and bound the UI design tokens directly to the backend properties so the UI reacts instantly.
